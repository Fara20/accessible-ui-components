**Accessible UI Component Library**<br>
Demonstrating Inclusive Design Principles for Web Interfaces<br>


**Project Goal**<br>
Create a set of reusable, accessible UI components that demonstrate user-centered design principles.<br>
Each component prioritizes:<br>
**1**.Screen reader compatibility (ARIA labels, semantic HTML)<br>
**2**.Keyboard navigation (no mouse required)<br>
**3**.Visual accessibility (color contrast, focus indicators)<br>
**4**.Mobile responsiveness (touch-friendly, responsive design)<br>

This project addresses digital exclusion by ensuring web interfaces work for users with diverse abilities.<br>

 **Components Included**<br>
**1. Accessible Form Input**<br>

->Proper label association<br>
->Error state announcements<br>
->Help text for screen readers<br>
->Clear focus indicators<br>

**2. Keyboard-Navigable Dropdown**<br>

->Arrow key navigation<br>
->Enter/Space to select<br>
->ESC to close<br>
->ARIA roles for assistive tech<br>

**3. High-Contrast Toggle Button**<br>

->Visual state changes<br>
->ARIA pressed state<br>
->Color-blind friendly<br>
->Clear active/inactive states<br>

**4. Skip Navigation Link**<br>

->Allows keyboard users to bypass repetitive content<br>
->Visible on focus<br>
->Industry best practice for accessibility<br>

**5. Accessible Modal Dialog**

->Focus trap (keeps keyboard inside modal)<br>
->ESC key to close<br>
->Screen reader announcements<br>
->Prevents background scrolling<br>


**Accessibility Standards**<br>
All components meet WCAG 2.1 Level AA requirements:<br>
**Perceivable:** Content is available to all senses<br>
**Operable:** Interface can be used via keyboard<br>
**Understandable:** Clear labels and error messages<br>
**Robust:** Compatible with assistive technologies<br>
